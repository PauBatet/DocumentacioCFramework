Templates

The CFramework template engine performs fast, direct string substitution. It searches for .html files within the templates/ directory and replaces tokens with dynamic content.

1. Template Syntax

Placeholders are defined using double curly braces {{token}}. There is no logic (if/loops) inside the HTML files; all logic is handled in C views. 

<!-- templates/profile.html -->
<header>
    <h1>{{name}}</h1>
    <p>{{title}}</p>
</header>

<div class="container">
    <section class="profile">
        <img src="{{image_url}}" alt="{{name}}">
        <p>{{about}}</p>
    </section>
    
    <section class="projects">
        {{projects}}
    </section>
</div>

2. Rendering from Views

Use the render_html function to send a template as an HTTP response. You pass an array of TemplateParam structures to map tokens to values. 

void profile_view(HTTPRequest *request, Database *db) {
    TemplateParam params[] = {
        {"name",      "John Doe",      NULL},
        {"title",     "C Developer",   NULL},
        {"image_url", "/static/j.jpg", NULL},
        {"year",      "2024",          NULL}
    };

    // Framework looks for "templates/profile.html"
    render_html(request, "profile.html", params, 4);
}

3. Advanced Template Orchestration

Complex layouts often require repeating elements or dynamic HTML generation. CFramework provides two ways to handle this.

Custom Field Converters

You can pass a C array to a template and provide a converter function that iterates through the data to return a single HTML string.

char* convert_projects(const void* value) {
    const struct Project* p = (const struct Project*)value;
    char* buffer = malloc(4096); // Allocated buffer
    buffer[0] = '\0';
    
    for (int i = 0; i < 3; i++) {
        char item_html[1024];
        snprintf(item_html, sizeof(item_html), "<h3>%s</h3>", p[i].title);
        strcat(buffer, item_html);
    }
    return buffer; // Framework handles free() after injection
}

Partial Processing

Use process_html to render small HTML fragments (partials) and concatenate them into a single token for the main template.

char *jobs_html = malloc(8192);
jobs_html[0] = '\0';

for (int i = 0; i < job_count; i++) {
    TemplateParam p[] = { {"title", jobs[i].title, NULL} };
    strcat(jobs_html, process_html("job_card.html", p, 1));
}

TemplateParam main_p[] = { {"jobs_list", jobs_html, NULL} };
render_html(request, "dashboard.html", main_p, 1);
free(jobs_html);
Design Philosophy: By keeping logic out of the templates, CFramework ensures that the "View" layer remains strictly in C. This maximizes performance and forces a clean separation between data preparation and presentation.
