Routing

Routes in CFramework define the relationship between a URL path and a C function. They are managed centrally in routes.c.

Defining Routes

The routing system uses a simple array of Route structures. Each entry contains a path string and the name of the view function it triggers. 

// routes.c
#include "views.c"

Route routes[] = {
    {"/", home},
    {"/wait", wait},
    {"/example", example}
};

Dynamic Parameters

To capture variables from a URL, use the <PARAM> syntax. These captured values are passed to your view function and can be retrieved by name. 

{"/user/<DNI>/profile", create_user_view}

In the example above, if a user visits /user/12345678X/profile, the framework captures "12345678X" as the parameter DNI.

Accessing Parameters in Views

Inside your view function, use the HTTPRequest_get_param helper to retrieve the value.

void create_user_view(HTTPRequest *request, Database *db) {
    const char *dni = HTTPRequest_get_param(request, "DNI");
    // 'dni' now holds "12345678X"
}
