Views

Views in CFramework are the bridge between raw HTTP requests and your database. They handle data extraction, business logic execution, and template orchestration.

Anatomy of a View

Every view function must match the framework signature: it takes a pointer to an HTTPRequest and a Database connection. 

void my_view(HTTPRequest *request, Database *db) {
    // 1. Process Logic
    // 2. Interact with Database
    // 3. Send Response
}

1. Handling Route Parameters

Data from dynamic routes is extracted using HTTPRequest_get_param. This provides immediate access to identifiers captured in routes.c. 

// In views.c
void create_user_view(HTTPRequest *request, Database *db) {
    const char *dni_param = HTTPRequest_get_param(request, "DNI");
    // Result: "12345678X" (if requested /user/12345678X/profile)
}

2. Database Lifecycle: Read, Modify, Update

A typical view involves fetching an existing record based on route parameters, modifying its fields in memory, and committing those changes back to the database. 

void update_user_age_view(HTTPRequest *request, Database *db) {
    const char *dni_param = HTTPRequest_get_param(request, "DNI");
    User db_user = {0};

    // 1. Fetch record from DB
    if (User_read(db, (char *)dni_param, &db_user)) {
        
        // 2. Modify memory-resident struct
        db_user.age = 31; 

        // 3. Persist changes back to DB
        User_update(db, (char *)dni_param, &db_user);

        // 4. CRITICAL: Free strdup'ed strings returned by User_read
        User_free(&db_user);
    }
}
Memory Management: Generated _read functions allocate memory for all string fields using strdup. Failing to call the generated_free function before the view finishes will result in memory leaks.

3. Advanced Template Orchestration

Complex layouts often require repeating elements or dynamic HTML generation. CFramework provides two ways to handle this.

Custom Field Converters

You can pass a C array to a template and provide a converter function that iterates through the data to return a single HTML string.

char* convert_projects(const void* value) {
    const struct Project* p = (const struct Project*)value;
    char* buffer = malloc(4096); // Allocated buffer
    buffer[0] = '\0';
    
    for (int i = 0; i < 3; i++) {
        char item_html[1024];
        snprintf(item_html, sizeof(item_html), "<h3>%s</h3>", p[i].title);
        strcat(buffer, item_html);
    }
    return buffer; // Framework handles free() after injection
}

Partial Processing

Use process_html to render small HTML fragments (partials) and concatenate them into a single token for the main template.

char *jobs_html = malloc(8192);
jobs_html[0] = '\0';

for (int i = 0; i < job_count; i++) {
    TemplateParam p[] = { {"title", jobs[i].title, NULL} };
    strcat(jobs_html, process_html("job_card.html", p, 1));
}

TemplateParam main_p[] = { {"jobs_list", jobs_html, NULL} };
render_html(request, "dashboard.html", main_p, 1);
free(jobs_html);
